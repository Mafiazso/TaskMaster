# บันทึกปัญหาโค้ดพังและวิธีแก้งาน (Error Log)

## 1. ปัญหาตารางเบี้ยว คอลัมน์สลับผิดช่องจน Error Index
**Error ที่เจอจาก Console:**
```text
Exception in thread "AWT-EventQueue-0" java.lang.ArrayIndexOutOfBoundsException: 6 >= 6
    at java.desktop/javax.swing.table.DefaultTableModel.getValueAt(DefaultTableModel.java:650)
    at com.pomodoro.ui.TaskTableModel.getValueAt(TaskTableModel.java:42)
```
**อาการตอนที่เจอ:** พอกดเพิ่มงานลงไป ข้อมูลมันลอยไปโผล่ผิดช่องหมดเลย เช่น ความสำคัญของงานดันกระโดดไปอยู่ตรงคอลัมน์คำอธิบาย พอกดดูคอลัมน์ที่ 7 (สถานะ) แอปก็เด้ง error รัวๆ
**สาเหตุ:** ตอนสร้างตาราง `TaskTableModel` กำหนดหัวคอลัมน์ไว้ 7 แถว แต่ดันลืมโยนค่า `Status` เข้าไปใน row ตอน return array ค่ามันมีแค่ 6 ช่อง พอดึงคอลัมน์ที่ 7 มันเลย Out of bounds
**วิธีแก้:** ไปแก้บรรทัดข้อมูลที่ return กลับใน `TaskTableModel` ให้มันส่งค่า `Status` เข้าไปด้วยจนครบ 7 ช่อง

---

## 2. เปิดโปรแกรมหน้าแรกแล้วตารางงานว่างโล่ง
**Error ที่เจอ:** ล็อกไม่ได้ขว้าง Error ออกมา แต่ตารางมันโล่งแปลกๆ ไม่เหมือนข้อมูลจริงใน DB 
**อาการตอนที่เจอ:** ในฐานข้อมูลเรามีงานจดคาไว้อยู่แล้ว ลองเช็ค sqlite ปรากฏข้อมูลก็อยู่ครบ แต่พอรันหน้าแอปขึ้นมาวันใหม่ กลับไม่เห็นข้อมูลอะไรโชว์ในตารางหลักเลย
**สาเหตุ:** สร้างตัวตาราง `TaskTableModel` ขึ้นมาวางไว้ประกอบหน้า UI เสร็จสรรพ แต่ไม่ได้สั่ง method ให้มันไปควานหาอ่าน data จาก database ตั้งแต่เพิ่งเปิดหน้าจอ
**วิธีแก้:** เติมบรรทัด `tableModel.reloadTable()` ต่อท้ายบรรทัดวาดตารางตัวโครงสร้างหน้าต่าง พอโปรแกรมมันเปิดขึ้นมา มันจะได้ไปโหลดหน้าตารางอัปเดตงานให้ทันที

---

## 3. กู้คืนงานแล้วงานหายทั้งสองตาราง
**Error ที่เจอ:** ข้อมูลไม่ sync กัน
**อาการตอนที่เจอ:** กดปุ่มลบงาน หน้าตารางหลักงานก็หายไปตามปกตินะ แต่พอจิ้มสลับไปดูหน้า Menu ของถังขยะดันไม่มีงานขึ้น พอลองพยายามกู้คืนงานกลับมา หน้า Menu งานก็ดันไม่โชว์งานคืนมาเหมือนกัน
**สาเหตุ:** ดันไปประกาศ `new TaskManager()` สร้างทิ้งไว้แยกกันในทั้งสองคลาส (`taskMenuDesign` เล่มนึง `trashMenuDesign` เล่มนึง) เวลาแอดหรือลบมันเลยไปแก้ตัวจัดการคนละชุด ข้อมูลเลยไม่เห็นตรงกัน
**วิธีแก้:** แก้ให้สร้าง `TaskManager` แค่ตัวเดียวตรงหน้าต่างหลัก `MainWindowDesign` แล้วส่งผ่าน Parameter ไปหาคลาสลูกทั้งสองคลาสแทน ให้ใช้งานร่วมกัน

## 4. รันโปรแกรมไม่ขึ้น หาไฟล์ Database ไม่เจอ
**Error ที่เจอจาก Console:**
```text
java.sql.SQLException: No suitable driver found for jdbc:sqlite:database/pomodoro.db
    at java.sql/java.sql.DriverManager.getConnection(DriverManager.java:706)
    at com.pomodoro.model.DatabaseManager.getConnection(DatabaseManager.java:12)
```
**อาการตอนที่เจอ:** กดรันแอปบนหน้าต่าง VS Code แล้วจอดำเด้ง error แดงเถือกใน Console โปรแกรมแตกตั้งแต่ต้น ยื่นเข้าหน้าหลักไม่ได้
**สาเหตุ:** ไปสั่ง import ไลบรารีคนละเวอร์ชัน ในตะกร้าไลบรารี (`lib/`) เรามีไฟล์ `sqlite-jdbc-3.50.3.0.jar` แต่ในไฟล์รันคำสั่งโครงงานดันไปใช้เลข `3.47.1.0` ทับซ้อนกัน มันเลยร้องหาไดรฟ์เวอร์ไม่เจอ
**วิธีแก้:** ขยับเลขเวอร์ชันบนโค้ดใน `pom.xml` ให้ตรงกับไฟล์ที่มีรันอยู่จริง เปลี่ยนให้เป็น 3.50 เหมือนกันทั้งหมด

## 5. ฟอร์มกรอกงานเด้งค้าง กดบันทึกไม่ไปกดยกเลิกไม่ได้
**Error ที่เจอ:** UI Freeze / Action Event ไม่ตอบสนอง
**อาการตอนที่เจอ:** พอกดปุ่มเพิ่มงานใหม่ กรอกทุกอย่างเสร็จปุ๊บ แต่พอเอานิ้วกดเมาส์ลงจิ้มคำว่า "บันทึก" หรือ "ยกเลิก" หน้าต่างมันนิ่งสนิทมาก ไม่ปิดหนีตัวไปเลย
**สาเหตุ:** วาดสร้างปุ่ม Jbutton สวยๆขึ้นมา แต่ดันลืมผูก `addActionListener` รับการกดให้มัน พอคลิกไปมันเลยไม่มีชุดคำสั่งรับไม้ต่อมาแรนเดอร์หน้าต่างปิดกั้น
**วิธีแก้:** ไปผูก Event ยัดใส่ปุ่มทั้งสองปุ่ม ถ้ากดเซฟค่อยหยอดงานลง DB แล้วรัน `dispose()` ถ้ากดยกเลิกก็ให้รันแค่ `dispose()` ยุบหน้าต่างเพียวๆ ทิ้งไป

## 6. ปุ่มเลือกเวลา Pomodoro กดติ๊กถูกพร้อมกันได้หมดทุกช่อง
**Error ที่เจอ:** Radio Button Multiple Selection Issue
**อาการตอนที่เจอ:** หน้าต่าง Setting ตรงส่วนที่ให้ตัวเลือกเวลาปุ่มกลมๆ (25:5 กับ 50:10) ดันสามารถกดติ๊กจุดติดดำได้พร้อมกันทุกข้อ
**สาเหตุ:** การวาด `JRadioButton` มันจะเป็นเอกเทศของใครของมัน ยอมให้หน้า UI กดจิ้มไปพร้อมกันได้ถ้าไม่มีใครมาช่วยห่อห้ามรวมกันไว้
**วิธีแก้:** ต้องสร้างกรงขึ้นมาชื่อว่า `ButtonGroup` ตัวนึง แล้วจับปุ่ม Radio ยัดพวกมันเข้าไปอยู่ในรวบเดียวกัน มันถึงจะรู้ตัวแล้วยอมเด้งออกสลับกันได้

## 7. กดคลิกขวาตารางไม่ทำงานเฉพาะบน Mac
**Error ที่เจอ:** PopupMenu ไม่โผล่ Mouse Event โดนทับ
**อาการตอนที่เจอ:** พยายามกดคลิกเมาส์ขวาตรงส่วนที่พ่วงเมนูในตารางงานเอาไว้ แต่หน้าจอนิ่ง บนตารางมันไม่โผล่ป๊อปอัพอะไรขึ้นมาเลย
**สาเหตุ:** ประกาศ `TaskTableModel` ตัวมันสืบทอดมาจาก `JScrollPane` (เป็นแผ่นคลุมตารางเลื่อนได้ในตัว) อยู่แล้ว แต่ดันทะลึ่งไปสร้าง `JScrollPane` ใหม่มาคลุมลูกทับมันอีกชั้นนึง กรอบของหน้าต่างมันเลยโดนห่อสองชั้น ตัวตรวจจับเมาส์มันเจาะไม่ทะลุแผ่นแรก
**วิธีแก้:** ถอด `JScrollPane` กรอบนอกสุดที่ห่อคลาสโมเดลนี้ทิ้งซะ แล้วยัดแอดโมเดลตรงๆ ใส่พาเนลไปเลย เมนูกลับมาคลิกโผล่ตามปกติ

---

## 8. เด้งเตือนหมดเขตงานตั้งแต่เริ่มวันใหม่
**Error ที่เจอ:** Date Comparison Logic Error
**อาการตอนที่เจอ:** กะว่างานจะส่ง "วันนี้" แบบสิ้นวัน แต่พอเข้าวันใหม่ตีศูนย์เป๊ะ เด้ง Alert ขึ้นมาฟ้องเต็มจอเลยว่างานก้อนนี้เลยวันครบกำหนดซะแล้ว
**สาเหตุ:** เวลาเราไม่ได้กรอกเวลาลง Calendar ของ Java ค่ามันจะถูกปัดกรอก default ไว้เป็น `00:00:00` พอรันเช็คเอาเวลา `new Date()` (ปัจจุบัน) มาเช็คว่าเวลาวันนี้มันล่วงหน้าเวลาในฐานไปแล้วรึเปล่า มันเลยร้องหาเรื่องทันที เพราะเที่ยงวันก็มองว่าเกินเที่ยงคืนไปแล้ว
**วิธีแก้:** บังคับปัดชั่วโมงวันเส้นตายงานทั้งหมดให้กลายเป็นเลข `23:59:59`

## 9. เด้งแครชเพราะพิมพ์ชื่อสถานะงานผิด
**Error ที่เจอจาก Console:**
```text
java.lang.IllegalArgumentException: No enum constant com.pomodoro.model.TaskStatus.TRUSH
    at java.base/java.lang.Enum.valueOf(Enum.java:273)
    at com.pomodoro.model.TaskStatus.valueOf(TaskStatus.java:3)
```
**อาการตอนที่เจอ:** พอกดสลับหมวดเปิดหน้าต่าง ถังขยะ แล้วโปรแกรมเด้งดุ้ง ปิดแครชไปเลย
**สาเหตุ:** ไปเผลอพิมพ์ภาษาอังกฤษคำว่าสถานะถังขยะว่า "TRUSH" พิมพ์บนสวิตช์เอนัม (`TaskStatus`) อย่างตั้งอกตั้งใจ แล้วพอหน้าตารางมันดึงคำสั่งโหลดคำว่าฐานมาเทียบหา มันบอกว่าไม่รู้จักคำบ้าบอนี้ในหัวตารางตัวเลือก
**วิธีแก้:** ตามเปลี่ยนและไล่ลบแก้ชื่อสถานะ "TRUSH" บนโค้ดรหัสของทุกไฟล์ เปลี่ยนกลับมาเป็นคำออริจินัลเขียนกติกาว่า "TRASH" ให้ถูกไวยากรณ์ แล้วหลบเพิ่มระบบดักอ่านในหน้าเก่าเผื่อฐานข้อมูลที่พังไปแล้วหลงเหลือด้วย

## 10. หน้า Home ไม่อัปเดตเมื่อมีคำสั่งเปลี่ยนงาน
**Error ที่เจอ:** Data Binding ขาดตอน ข้อมูลหน้าเก่าไม่อัปเดตให้ใหม่เมื่อกลับมาจากหน้าอื่น 
**อาการตอนที่เจอ:** เข้าไปเพิ่มงาน หรือสั่งลบงานย้ายเข้าถังขยะจากหน้า Task แต่พอกดกลับมาเมนูหน้า Home สถิติรวมและตารางกลับนิ่ง ไม่มีการขยับตัวตามที่เพิ่งเพิ่ม/ลบไป
**สาเหตุ:** ไม่ได้ทำคำสั่งรีโหลดหน้า Home พ่วงกับจังหวะที่เซฟงานลงฐานข้อมูล หรือจังหวะที่กดปุ่มสลับหน้า ทำให้หน้าค้างอยู่กับข้อมูลชุดแรกที่ดึงตอนรันเปิดโปรแกรม
**วิธีแก้:** สร้างฟังก์ชัน `reloadHome()` แล้วเอาไปผูกโยงกับปุ่มคลิกกดสลับหน้าตรงเมนูซ้ายมือ (Sidebar) พอกดคลิกสลับมาปุ๊บก็สั่งเคลียร์รีโหลดดึงตารางซ้ำตลอดเวลา

## 11. แจ้งเตือนเวลาไม่ทำงานหลังจากเปิดแอปค้างไว้ และแก้ไขเวลาเตือนไม่ได้
**Error ที่เจอ:** Logic Design เช็คแจ้งเตือนทำงานแค่ 1 ครั้ง และติดตัวจำรหัสเดิม (Cache)
**อาการตอนที่เจอ:** ระบบแจ้งเตือนเวลาเดดไลน์ มีป๊อบอัปขึ้นจริงแต่เฉพาะตอนผู้เล่นกดเปิดโปรแกรมประเดิมช่วงแรก แต่ถ้าเปิดแอปทิ้งค้างจอไว้ ระบบจะเงียบกริบ แม้จะถึงเวลาเดดไลน์แล้ว นอกจากนี้ถึงผู้เล่นพยายามแก้เวลา Reminder Time ใหม่เพื่อเทส ระบบก็จำรหัสเดิมว่าเคยแจ้งแล้ว เลยไม่ยอมเตือนซ้ำ
**สาเหตุ:** โค้ดที่เรียกใช้คำสั่งเช็คเวลาถูกผูกติดฝังไว้ตอนวาดการ์ดหน้าแรกแค่รอบเดียว ไม่มีลูปคำสั่งที่คอยไล่เช็คเวลาอยู่ตลอดในพื้นหลัง และตัวช่วยจำว่าแจ้งอะไรไปแล้วบ้าง ดันจำแค่ ID งานอย่างเดียว
**วิธีแก้:** ใช้รันระบบเวลา `javax.swing.Timer` ในพื้นหลัง ตั้งเวลาทุกๆ 5 วินาทีให้วนลูปตรวจสอบแบบถี่ยิบตลอดทั้ง session และเปลี่ยนตัวแปรช่วยจำให้เซฟรวมทั้ง ID งาน + ขีดเส้นใต้ + เวลา Reminder ชุดนั้นๆ เผื่อกรณีมีการกลับมาบันทึกแก้เวลาจะได้เด้งเตือนใหม่ได้

## 12. เปิดโปรแกรมหน้าแรกดันไม่ใช่หน้าตั้งต้น
**Error ที่เจอ:** ลำดับการเรียงสลับหน้า CardLayout ผิด
**อาการตอนที่เจอ:** เปิดตัวโปรแกรมขึ้นมาแล้วมันดันเด้งโผล่ที่หน้า Task แทนที่จะเป็นหน้า Home เหมือนที่โปรแกรมทั่วไปทำ 
**สาเหตุ:** จังหวะเรนเดอร์โครงหลักใน `MainWindowDesign` โยนหน้า Task ใส่กระดานเพิ่มลงไปใน CardLayout เป็นชิ้นแรก มันเลยถือว่าชิ้นนั้นเป็นการ์ดเบอร์หนึ่ง 
**วิธีแก้:** เรียงคำสั่ง `cardLayout.show(..., "HOME")` ประกบปิดท้ายก่อนจบชุดคำสั่งเตรียมแสดงหน้าต่างหลัก เพื่อให้ไพ่ที่หงายขึ้นเป็นหน้า Home เสมอ
